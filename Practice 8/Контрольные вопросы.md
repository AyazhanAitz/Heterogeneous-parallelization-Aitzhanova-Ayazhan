## **1. Какие преимущества предоставляют гибридные вычисления?**

Гибридные вычисления позволяют **одновременно использовать CPU и GPU**, что обеспечивает:

* более эффективное использование вычислительных ресурсов;
* сокращение общего времени выполнения за счёт **параллельной работы** CPU и GPU;
* возможность перекрытия вычислений с передачей данных;
* повышение производительности для задач с большим объёмом данных и высокой степенью параллелизма.

## **2. Как минимизировать накладные расходы при передаче данных между CPU и GPU?**

Накладные расходы можно уменьшить следующими способами:

* использовать **асинхронную передачу данных** (`cudaMemcpyAsync`);
* применять **CUDA streams** для перекрытия копирования и вычислений;
* минимизировать количество копирований между CPU и GPU;
* использовать **pinned (page-locked) память**;
* по возможности хранить данные на GPU между несколькими вычислениями.

## **3. Какие задачи лучше выполнять на CPU, а какие — на GPU?**

**На CPU целесообразно выполнять:**

* задачи с последовательной логикой и ветвлениями;
* управление потоками, ввод-вывод, подготовку данных;
* задачи с небольшими объёмами данных.

**На GPU целесообразно выполнять:**

* задачи с высокой степенью параллелизма;
* однотипные операции над большими массивами данных;
* вычислительно интенсивные задачи (линейная алгебра, фильтрация, моделирование).

## **4. Как можно улучшить производительность гибридного приложения?**

Производительность гибридного приложения можно повысить за счёт:

* сбалансированного распределения нагрузки между CPU и GPU;
* использования **асинхронных вычислений и streams**;
* оптимального выбора размера блоков и сетки CUDA;
* уменьшения количества обращений к глобальной памяти;
* перекрытия вычислений CPU с вычислениями и копированием данных на GPU.
